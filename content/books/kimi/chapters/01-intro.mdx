<SectionTitle number="II"><T zh="它长成了一套对标前沿的栈" /></SectionTitle>

<T zh={<>把它的产品线摆出来, 几乎能跟 OpenAI 和 Anthropic 一一对上。旗舰模型 K3 是 2.8 万亿参数的 MoE、1M token 上下文, 原生多模态、思考永远开着, 权重官方承诺 7 月 27 日前放出。<Footnote n={2} />往上按官方的摆法是五件套: 网页里的 Kimi 本体（聊天加五档会员, 外带 Agent 做成品、Agent Swarm、Deep Research、Kimi Claw 一排入口）、桌面端的 Kimi Work、终端里的 Kimi Code、浏览器扩展 Kimi WebBridge, 和最底下的 Kimi Platform 开放平台 API。<Footnote n={1} /></>} />

<T zh={<>「一一对上」只证明形态, 证明不了它们是一体的。让我改口叫它「栈」的, 是两层咬合。<strong>同一颗脑子</strong>: 四模式、Agent、Swarm、Kimi Code 背后是同一条模型线 —— K3 一发布, 官方把 Agent 集群也切到它驱动, 没有各产品各养各的模型。<Footnote n={6} /><strong>同一本账</strong>: 一个会员账号串起所有入口, 聊天之外的成品、研究、集群共享一个 token 额度池 —— 第九章会算到, Swarm 跑得猛, 是真的会把 Agent 的额度吃紧。<Footnote n={7} />一颗脑子加一本账, 动一层、其他层跟着动, 这是「栈」和「杂货铺」的区别。丑话也说在这: 生态那一层还薄 —— 插件、第三方集成、企业采购的成熟度, 都还追不上那两家。所以本书说「对标」, 说的是产品形态和模型能力两层; 生态这层, 先不算数。</>} />

<T zh={<>最能说明它换了打法的, 是价格。两年三级跳: K2.5 输入每百万 token ¥4.00, K2.6 涨到 ¥6.50, K3 直接定到 ¥20.00、输出 ¥100.00。<Footnote n={3} /><Footnote n={4} />一个还在抢量的便宜货不会这样定价。但话要说准: 价格证明的是它的自我定位, 证明不了它值这个钱 —— 值不值, 要看单位任务成本和独立实测, 那是第三章一笔一笔算的账。<strong>抢量、拼便宜那条路, 现在是 DeepSeek 在走; Kimi 把自己摆上了对标前沿的货架</strong> —— 摆没摆稳, 价格说了不算, 后面一格一格验。</>} />

<SectionTitle number="III"><T zh="这本书给这套栈画一张图" /></SectionTitle>

<T zh={<>这本书不列全部功能, 也不教点哪个按钮（界面几个月一变, 判断不会）。它给这张地图标的是: 每一面<strong>对标前沿的哪一个、省什么、什么时候用、什么时候别用</strong>。后面顺着这套栈走 —— 全景、挑模型、四种模式、Agent 交办、Agent Swarm 与 Claw、Deep Research、Kimi Code 与 API, 然后是选档与取舍, 末尾一页实战速查。</>} />

<T zh={<>也把话说在前头: 「长成全栈」不等于「样样第一」。我说的「差前沿半步」, 点名道姓、带上刻度: 最难的推理、最吃打磨的 agentic coding、一锤定音的输出, 地盘仍归 Claude Fable 5 和 GPT 5.6 Sol —— 独立榜 DeepSWE（2026-07-17）上 K3 是 69%、榜首 73%, 差 4 个点、误差棒还重叠; 官方在发布博客里自己也承认「trails the most powerful proprietary models」。<Footnote n={5} /><Footnote n={2} />「半步」就是这个距离: 看得见, 还没追上。本书每一面都会标出这条线, 该回 frontier 就回。还有一条最容易踩的: <strong>会员订阅不含 API</strong>, 开放平台（platform.kimi.com）是另一套账户、另一条账单。<Footnote n={3} />它们重要, 只是别混成一笔账。</>} />

<Callout kind="note" zh={<>立场说在前面: 我拿自己的钱用它, 没拿 Moonshot 的钱。官网自己能写的话这里不写; 不接 affiliate, 推荐一样东西会说清为什么, 也会说清它的代价。模型版本、价格、上下文长度都标了日期 —— Kimi 迭代很快（这一年从 K2 一路走到 K3）, 看到本书时请以官网当时的页面为准。</>} />

<T zh="动笔之前, 我先照了一次镜子。翻自己两周的 Kimi 记录, 数「有几件不是默认聊天」—— 数出来是 1。这个数本身没意义: 多点开几个入口不等于把活干好, 它量的是「用没用」, 不是「有没有用」。我改成记三笔账: 哪件事它替我省了时间, 哪件原来做不成的做成了, 哪件质量明显上去了。三笔都记不出来的那一面, 对我仍是摆设。后面九章, 每一章都是在替我回答: 这面能不能记上账。" />

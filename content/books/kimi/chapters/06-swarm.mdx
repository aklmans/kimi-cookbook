<T zh="一个 agent 一次只能顺着一条线干。可有些活不是「难」, 是「多」—— 要翻一百个网页、要把四十份 PDF 读成一篇综述、要同时从十个角度看一个问题。这种活, 一个 agent 排队干到天黑, Kimi 干脆放一群上去。" />

<T zh={<>这一章讲两条「大活」的线: <strong>Agent Swarm</strong>（用规模换时间, 一次派出一群子 agent 并行）和 <strong>Kimi Claw</strong>（用自主换托管, 一个 7×24 替你动手的常驻 agent）。两条都猛, 也都各有一笔你必须先认清的代价。</>} />

<SectionTitle number="I"><T zh="Agent Swarm 换的是「串行排队」的时间" /></SectionTitle>

<T zh="它的本事不是更聪明, 是更多、更并行 —— 把一件能拆开的大活, 同时摊给几百个分身去干。" />

<T zh={<>到 K3 时代, Agent Swarm 改由 K3 驱动（官方叫「K3 集群」）, 一次任务最多能派出约 <strong>300 个子智能体</strong>同时工作、跑超过 <strong>4,000 次工具调用</strong>, 比单个 agent 一步步串着干约快 4.5 倍 —— 这组数字还是 K2.6 时代那组, 官方没随 K3 更新。<Footnote n={1} />它扛的是「规模型」的活: 大规模检索、批量处理上百份文档、产出十万字以上的报告、复杂编程。给个对照: 单 Agent 是一个人加班到天亮, Agent Swarm 是一支临时组起的小队, 分头干完再归并。</>} />

<KimiSwarmDiagram />

<T zh={<>但规模是有标价的。Agent 集群按档限次、且<strong>消耗约为普通 Agent 任务的数倍</strong>, 目前仍带 Beta 说明, Moderato（¥99/月）档起有 25 次, Allegretto 50 次、Allegro 120 次 —— 门槛比本书上一版写的时候低了一档。<Footnote n={1} /><Footnote n={3} />所以它不是「更高级的默认」, 是一件你确认值得才动用的重型工具。</>} />

<Divider />

<SectionTitle number="II"><T zh="它只对「拆得开、并得了」的活值" /></SectionTitle>

<T zh="判断要不要上 Swarm, 只看一件事: 这活能不能拆成一堆互不依赖、可以同时干的小块。" />

<T zh={<>Swarm 的甜区是三种「规模」: 一是<strong>查得广</strong>（把散在上百处的信息一次捞齐）, 二是<strong>产得多</strong>（一口气读完几十份材料、写成一份长报告）, 三是<strong>看得全</strong>（同时让多个分身从不同角度看一个问题, 逼出分歧而不是抱团附和）。这三种活的共同点是: 能切成并行的块。<Footnote n={1} /></>} />

<T zh="反过来, 一件必须一步接一步、后一步等前一步结果的线性活, 给它一支 swarm 毫无意义 —— 分身们彼此干等, 你却付着数倍的额度。一个人能顺手办完的事, 别叫一支小队来开会。先问「这活拆得开吗」, 拆不开, 留在单 Agent 甚至对话里更省。" />

<T zh="举个值得动用 swarm 的真实场景: 一周内摸清一个新赛道 —— 几十家公司的定价、定位、最近融资。一个 agent 串着查要查到天黑, 而这种「查得广 + 产得多」正好拆得开。可以这样交代它:" />

<SectionTitle number="III"><T zh="Kimi Claw: 能力更大, 那条线也更紧" /></SectionTitle>

<T zh="Agent Swarm 是「一次干一票大的」, Kimi Claw 是「一直替你干」—— 自主度高一截, 你要守的线也紧一截。" />

<T zh={<>Kimi Claw 没有死，是搬了家: 入口从 /claw 迁到了 kimi.com/bot, 形态也从「云端常驻 agent」一个长成了一族 —— 云端 7×24、桌面端、安卓端三种部署, 加上微信 / 飞书 / 企微 / 钉钉 / 微博的 IM 机器人, 还有一个 Preview 状态的多 agent 协作空间「Claw 群聊」。7×24 常驻、40GB 云存储、ClawHub 5,000+ 技能、定时任务这些老本事都在, 内置 Kimi 模型、免你配 API。<Footnote n={2} />它能干的事, 比「做一份 PPT」大得多 —— 因为它不只动脑, 还常驻、联网、能替你动手。</>} />

<T zh={<>两处细节该知道: 一, Claw 本体的 Beta 角标摘了, 但新形态「Claw 群聊」仍标着 Preview —— 预发布的帽子是换了地方, 不是摘完了, 别当整条线已转正; 二, 默认模型官方两页说法不一（/bot 写 K2.6 Thinking、官方资源页写 K3), 以你部署时页面实际为准。档位要求没变, 仍是 Allegretto（¥199/月）起。<Footnote n={2} /><Footnote n={3} /></>} />

<Callout kind="warn" zh={<>但「常驻 + 联网 + 能动手」三样叠在一起, 意味着授权给到哪、数据落在哪、出了错谁兜底, 你都得先想清楚。截至 2026-07, 它的 Beta 角标摘了、新形态群聊仍标 Preview, 档位也仍要 Allegretto（¥199/月）以上。<Footnote n={3} />这是典型的「能用」不等于「现在该押上去」: 普通用户先把前面的 Agent 交办用熟, 这条线等你想清边界再碰也不迟。<strong>能替你 24 小时动手的 agent, 也能 24 小时替你闯祸。</strong></>} />

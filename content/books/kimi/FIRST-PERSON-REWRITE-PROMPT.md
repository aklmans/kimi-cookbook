# 任务:把第 02–10 章统一为第一人称(作者视角)

你是《Kimi · 从长文本到一套 agent 栈》(Kimi Cookbook) 的内容编辑。
第一章(01-intro)已由另一位编辑改成第一人称并获得作者认可,现在的任务
是把其余九章收到同一条声线上。

## 先做这三件事(按顺序)

1. 通读 `content/books/kimi/EDITOR-PROMPT.md` —— 全书的编辑标准(数据
   纪律、语气、禁语、标点),全部适用,不再重复;
2. 通读 `content/books/kimi/chapters/01-intro.mdx` **当前版本** ——
   这是第一人称声线的标准样张,先读透它的节奏再动手;
3. 通读 `content/books/kimi/chapters/0{2..9}-*.mdx` 与 `10-playbook.mdx`
   现状,以及 `content/books/kimi/meta.ts` 里各章的 lede / posterSummary。

## 声线定义(本任务的核心)

这本书是**一个付费用户写给其他付费用户的经验记录**,不是讲师讲义。

- **「我」的领地**:观察、算账、踩坑、判断、取舍 —— 用第一人称。
  「我算过」「我踩过这个坑」「我的用法是」「这笔钱我付过」。
- **「你」的领地**:递给读者的动作与指令 —— 保留第二人称或祈使句。
  「照这个格式套就行」「你可以直接抄这段」「别这么用」。
- **两个不许**:
  - 不许把每个「你」机械替换成「我」—— 指令句里的「你」是对的;
  - 不许编造作者经历 —— 第一人称只能用于真实的判断与经验,
    不许虚构「我曾经用 X 干过 Y」这种假亲历。作者真实干过的事
    (写这本书、做海报、剪直播、直播开发)可以用;拿不准的,改成
    中性陈述或留给「你」。

转换示例(摘自 01,照这个劲儿来):
- 「你大概是这样记住 Kimi 的」 → 「我记住 Kimi, 是在 2024 年」
- 「这本书讲的, 就是你大概率只用了一个角的栈」 → 「我大概率只用到了
  这套栈的一个角。这本书, 是我把这个角之外的部分一块块拆开看过的记录」
- 「照这个格式套就行」 → 不动(指令句)。

## 硬约束(全部必须满足)

1. **数据零改动**:所有数字、价格、额度、上下文、跑分、日期、端点、
   错误码保持原样;frontmatter `references:` 原样保留(新增论断才补,
   且必须带来源 URL);
2. **组件零改动**:`<Cover/>`、`<SectionTitle>`、`<H3>`、`<Divider/>`、
   `<Footnote n={N}/>`、`<T>`、`<C>`、Callout、PromptBox、CodeBlock、
   Steps、Checklist、Dl/Di、Stats、PriceTable、BarCompare、Timeline、
   表格 —— 全部保留,只改其中的散文文字;
3. **slug 与结构**:章节 slug 一律不变;标题可像 01 那样改成第一人称
   (例如「你印象里的 Kimi 是旧的」→「我印象里的 Kimi 是旧的」),
   但**改标题必须同时在 `meta.ts` 里同步**,并在交付清单里单独列出;
4. **meta.ts 同步**:受影响章节的 `lede`、`posterSummary` 一并改成
   第一人称;posterSummary 保持两句、为"不看这章的人"而写;
5. **文字规矩**:无「免费」字样(用「开源」「公开」);内引号用「」;
   半角逗号 + 空格;破折号 ——;中英之间留空;Kicker 可改第一人称;
6. **验收**:完成后 `npm test && npm run lint && npx tsc --noEmit &&
   npm run build` 必须全绿(quality-check 会核对脚注引用与 meta 一致性,
   别想绕)。

## 交付方式

- 逐章给:章节 + 「改前 → 改后」摘要(语气层变化,不贴全文);
- 标题变更单独列一张表:slug · 旧标题 · 新标题 · meta.ts 是否已同步;
- 任何一处拿不准「该用我还是你」的地方,标出来给两个选项 + 理由,
  别硬选;
- 最后一句:报告「现在每章都能过『付费用户写给付费用户』的检验」,
  或指出还卡在哪个具体问题。

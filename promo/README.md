# Promo grammar · 小红书海报与视频制作手册

给后面接手的 Agent:这个目录的海报母版(`*.html` + `base.css`)和
`scripts/build-promo-video.mjs` 不是一次性产物,是一套**可复用的语法**。
照这份手册做,就能复现同等的视觉效果。所有判断都有出处 —— 每一处"为什么"
都是踩过坑之后定的。

---

## 1 · 总原则(两条,先于一切)

1. **海报和视频是网站的延伸,不是新设计**。色板、字体、月亮、排版全部来自
   Zhaphar's Design Language · v3(见 `DESIGN.md`)。不发明新颜色、不发明新字体。
2. **每一帧都要亲眼看**。渲染完必须:① 全尺寸看一遍;② 缩到信息流尺寸
   (375px 宽)再看一遍;③ 找"粘在一起"的地方(文字压文字、卡片压落款)。
   这三步缺一不可 —— 本目录的历史 bug(弹层压扁、QR 卡粘落款、序号被圆盘吃掉)
   全是漏做第③步放出去的。

---

## 2 · 海报语法(`01-cover.html` … `05-cta.html` + `base.css`)

### 画布与管线

- 1080×1440(小红书 3:4),`deviceScaleFactor: 2` 输出 2160×2880;
- 渲染:`node scripts/render-promo.mjs promo/XX.html output/promo/XX.png`
  (Playwright 截图,`omitBackground: true` —— 透明区真透明,
  这是视频字幕带中窗的前提);
- 字体用 `assets/fonts/` 的**全量** woff2(W04/W05),不用子集 ——
  海报文案是变量,子集字形集不覆盖。

### 骨架(系列统一,逐张只变内容)

```
┌ masthead:月亮 tile + KIMI COOKBOOK + NO. 0X
├ kicker:圆形 K 徽章 + 栏目标签(letterspaced 灰字)
├ 大标题:Tsanger 600,墨色,关键词 Kimi 蓝(#1783FF)
├ 内容区(逐张不同:封面钩子 / 目录表 / 截图组合 / 卡片格 / QR)
└ colophon:三端栏(在线书 / 网站应用 / 小程序)hairline 分隔
```

- **色板**(只许用这些):paper `#FAFAFA`、ink `#1A1A1A`、ink-2 `#3A3A3A`、
  ink-3 `#6B6B6B`、hairline `#C0BFBA`、card `#F0EFEB`、disc `#0E0E13`、
  ghost `#1B1B25`、crescent `#EFE8DC`、blue `#1783FF`;
- **月亮大圆盘**:直角满幅出血(不做圆角,平台自己会裁圆形),幽灵盘 +
  纸月牙 + 蓝卫星,**所有元素收在内切圆内**(微信圆裁不咬边);
- **图标纪律**:圆角方 tile = 品牌(刊头专属);圆形徽章 = 主题(kicker 的
  K 标)。两个方形图标不许上下摞 —— 这是用户点名的"两个 App 图标排队"事故;
- **期号**:`NO. 01`…`NO. 05`,刊头右侧,系列感靠它;
- **字号阶梯**:kicker 30 / h1 116(封面)或 62–76(内容页)/ sub 36 /
  正文 26–33 / 注脚 22–26。行高 1.32–1.35,CJK 行内英文不断词。

### 文案纪律

- 禁用"免费/free"字样(用户明令,引流味太重)—— 用"开源""公开可读";
- 小程序**永不出现名字**,统一"扫码体验 / 海报末页小程序码"(备案改名痛点);
- slogan `Think clearly. Build with Kimi.`(Kimi 蓝色加粗)**只放在
  呼吸得开的地方** —— 封面和视频首帧;内容页、QR 卡旁不放(会粘);
- 每张海报 = 一个钩子 + 一个信息层,别塞第二张脸。

## 3 · 视频语法(`scripts/build-promo-video.mjs`)

### 结构与时长

- 1080×1440 竖版,30fps,总长 60–75s,**无声**(字幕压死,BGM 发布时加);
- 四段式:**钩子卡(3s)→ 蒙太奇(13 段,2.5–4s/段,5–8× 加速)→
  成果实录(~8s,1×,线上真实滚动录屏)→ CTA 卡(4.5s)**;
- 蒙太奇段落从长时间录像里挑:先按 300s 间隔抽帧拼 contact sheet,
  **只挑高信息量时刻**(模拟器点亮、满屏红错、GitHub 推送、页面成型),
  绝不盲采样 —— 空转/静止帧是蒙太奇的天敌。

### 字幕带(每段一张,透明中窗)

- 上带:纸底,小 caps 栏题(26px)+ Tsanger 52px 段题;
- 下带:进度轨道 + 蓝色进度条(按段号递增)+ KIMI COOKBOOK + `NN / 13`;
- 中窗透明(靠 `omitBackground`),视频画面 1080×608 居中 y=416;
- **透明 bug 教训**:Chromium 截图默认垫白底,第一版蒙太奇全是白框 ——
  透不明白,先看 PNG 角落像素再批量。

### ffmpeg 管线(分段编码再拼接)

```
每段:trim(-ss/-t)→ setpts 加速 → scale=1080:608 → pad 进 1080×1440
     → overlay 该段字幕带 → libx264 yuv420p 30fps → concat demuxer -c copy
```

- 中间产物统一规格才能 `-c copy` 直拼;卡片用 `-loop 1 -t N` 转视频;
- 验收:`ffprobe` 看时长/尺寸/码率,抽 t=1 / 中段 / 尾段三帧亲眼看;
- 卡片(钩子/CTA)与海报同一套刊头、月亮、蓝。QR 卡与落款之间
  留 ≥60px 呼吸(粘过一次,别再犯)。

### BGM(发布侧)

- 首选钢琴轻音乐(石进《夜的钢琴曲五》),次选 lo-fi/chill beats;
  无人声或极少人声,音量 40–60%;
- 只在小红书/剪映曲库内选(平台授权),**不导入本地下载的版权曲**;
- 卡点:钩子卡弱起、蒙太奇进律动、成果段稍缓、CTA 收弱。

## 4 · 验收清单(做完物料,逐条打勾)

- [ ] 全尺寸 + 信息流尺寸(375px)各看一遍每张图
- [ ] 无任何粘连:文字压文字、卡片压卡片、图标被圆盘/边线吃掉
- [ ] QR 码在圆形裁剪下完整可扫,小程序名零出现
- [ ] 无"免费"字样,slogan 只在封面/首帧
- [ ] 视频:`ffprobe` 时长对、抽帧三色(卡/段/CTA)无白框无黑屏
- [ ] 新文案过一遍字形:海报用全量字体,网站改动后记得 `npm run gen:font`
